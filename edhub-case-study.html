EdHub: UX Case Study

01OVERVIEW

Challenge

Elementary teachers need a way to effectively give instructions and assignments with their students. How might we re-imagine the online learning platform for elementary teachers during COVID-19?

Objective

Develop an online platform for teachers to find pre-made assignments to cut down on preparation time for lesson plans and a way for teachers to communicate with families.

Solution

Elementary teachers need a way to effectively give instructions and assignments with their students. This solution will decrease amount of time spent on the learning platform because teachers need more time to focus on lesson planning to teach virtually.

My Role

UX designer designing a product for teachers from conception to delivery

Competitive Analysis

Gathering information about the competition helped me define the basic features of my application. I found that none of the platforms have parent accounts for effective communications (or parents would usually log into their child's account). In addition, one of the platforms allowed for the user to look and assign work created from other teachers which will cut down on finding assignments that align to the week's standards.

Site Map

Site mapping helps improve website navigation, understanding the scope of work, and avoid missing critical parts of a website’s architecture. During this process, I was able to determine how many screens I may need for my product.

User Flow

Creating a user flow helped form a route map and define all navigation steps in the user journey in order to achieve their goal. This user flow is based on assisting teachers find assignments that align with their lesson plans so they do not have to recreate assignments.

Design System

Low-Fidelity Wireframes

Low fidelity wireframes include the most basic content and visuals and are usually static. It helps to map out the shell of the interface, its screens, and basic information architecture.

Upcoming Design Changes

Dashboard changes are more personalized to the user’s preferences and defines a layout and flow.

Announcement cards are now created as a component with auto layout. Some typographies were changed such as line and letter spacing, and the color contrast. In addition, a kebab menu was added to eliminate multiple buttons.

Interacting with the pinned accouncements opens smoother with smart animate.

The new navigation menu separates the community and personal library into their own category for a simpler hierarchy of action. The icons are placed on the left side of the text label. The navigation menu was created as a component using auto layout.
